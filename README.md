# HTML and Git Practice by Tabitha Khadse

This project follows a six hour sequence that pairs HTML learning with Git workflow.  
Flow: Initialize to Commit to Branch to Merge to Manage Remotes.

## Hour 1 Git setup and HTML basics
HTML topics: what is HTML, headings, lists, basic page.  
Git: git init, git config, git status, git add, git commit.  
Goal: initialize repository, configure user, create index.html with headings and lists, first commit.

## Hour 2 HTML structure and history
HTML topics: attributes, hyperlinks, img element, id and class.  
Git: git log --oneline, git diff, git commit --amend, git reset --soft.  
Goal: add images and links with attributes, review history, amend a commit to refine message or include a forgotten file.

## Hour 3 Semantic layout and remotes
HTML topics: inline and block concepts, header, nav, main, footer.  
Git: git remote add origin URL, git branch -M main, git push -u origin main.  
Goal: refactor to semantic layout, connect to GitHub, push main.

## Hour 4 Data structures and branching
HTML topics: tables using table, tr, th, td.  
Git: git branch feature_table, git switch -c feature_table, git push --set-upstream origin feature_table.  
Goal: create a feature branch, add a table, commit and push the branch.

## Hour 5 User input and merging
HTML topics: forms using form, input, label, textarea.  
Git: git merge feature_table, resolve conflicts, git commit.  
Goal: add a contact form on the feature branch, make a conflicting change on main, merge and resolve.

## Hour 6 Cleanup and review
HTML topics: basic accessibility such as alt text and input labels.  
Git: git fetch, git pull, git stash, git branch -d.  
Goal: practice stash save and restore, delete the merged feature branch locally and remotely, final push and history review.

## Quick start
1. Clone or open the repo  
2. Open index.html in a browser
