About This Project

My name is Tabitha Khadse. This page documents my step by step practice combining HTML concepts with Git version control. Each section represents one phase of my learning plan while using Git to manage every change.

Learning Goals

  • Understand HTML structure and semantics
  • Learn core Git commands through practice
  • Integrate HTML development with version control
  • Practice commits, branching, merging, and remote management

6 Hour Integrated Git and HTML Plan

Hour Focus Topic HTML Concepts Git Commands Practical Goal and Output
1 Git Setup and HTML Basics What is HTML, headings, and lists. Creating basic pages. git init, git config, git status, git add, git commit Initialize repository, configure user, create index.html with headings and lists, first commit.
2 HTML Structure and History Attributes, hyperlinks, images, id, and class. git log --oneline, git diff, git commit --amend, git reset --soft Add images and links using attributes. View history and amend commits.
3 Semantic Layout and Remotes Inline vs block, header, nav, main, and footer. git remote add origin URL, git branch -M main, git push -u origin main Refactor to semantic layout and push project to GitHub.
4 Data Structures and Branching HTML tables (table, tr, th, td). git branch feature_table, git switch -c feature_table, git push --set-upstream origin feature_table Create a feature branch and add a complex HTML table.
5 User Input and Merging HTML forms (form, input, label, textarea). git merge feature_table, resolve conflicts, git commit Add a contact form, create a merge conflict, resolve it, and commit.
6 Cleanup and Review Accessibility such as alt text and labels. git fetch, git pull, git stash, git branch -d Use stash, delete merged branch, and finalize repository.
